## 自動更新觸發條件

當發生以下情況時，**必須**更新對應文件：

### 更新 README.md
- 新增或刪除重要目錄
- 變更 URL 結構
- 新增維護腳本

### 更新 GUIDELINES.md
- 新增或修改開發規範
- 變更命名慣例
- 更新 AI 協作準則

### 更新 CONTEXT.md
- 完成重大功能或修復
- 專案狀態或階段變更
- 新增待辦事項
- **每次更新必須同步修改「最後更新時間」**

### 更新 .agent/System/changelog.md
- `.agent/` 目錄結構變更
- `.claude/commands/` 新增或修改
- SOP 新增或重大修改
- 專案結構重構（如圖片搬移、目錄重組）
- 重要決策執行完成

### 更新 .agent/System/learnings.md
- 專案慣例或最佳實務有變更
- 發現新的避免事項
- 結構規範改變（如圖片路徑格式）

### 更新 .agent/System/decisions.md
- 做出重要架構或流程決策
- 記錄決策背景、選項與理由

## 維護流程

### Step 1：檢查變更範圍
```bash
# 列出近期變更的檔案
git status
git diff --stat HEAD~5
```

### Step 2：識別需更新的文件
- 新增目錄 → 更新 `README.md` 目錄結構
- 新增規範 → 更新 `GUIDELINES.md`
- 狀態變更 → 更新 `CONTEXT.md`

### Step 3：執行更新
- 只修改變動部分，不重寫整份文件
- 使用繁體中文撰寫
- 遵循 GitHub Flavored Markdown 格式

### Step 4：驗證一致性
- 確認三份文件間的交叉引用正確
- 確認 URL 結構與實際目錄對應
