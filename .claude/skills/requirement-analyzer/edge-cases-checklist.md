## 使用方式

進行需求分析時，根據功能類型選擇相關的檢查維度：

| 功能類型 | 主要關注維度 |
|----------|--------------|
| 表單功能 | 輸入邊界、網路邊界 |
| 列表功能 | 資料邊界、裝置邊界 |
| 會員功能 | 權限邊界、狀態邊界 |
| 上傳功能 | 輸入邊界、網路邊界 |

**不需要每個 edge case 都處理**，但應該：

1. **識別** — 知道有哪些可能情況
2. **評估** — 判斷發生機率和影響
3. **決策** — 決定處理或暫不處理
4. **記錄** — 標註「⚠️ 待確認」或「不處理，因為...」
